## 0. 一句话方向

> **在一只打盹的老龟面前问一卦——只问一句,他用一个字 + 一句人话回你。**

## 1. 与 v2 的核心差异(差异表)

| 维度 | v2.0 | v3.1 |
|---|---|---|
| 引擎 | 梅花易数 (单动爻) | **六爻 lite**(纳甲+五行+六亲+用神,多动爻) |
| 输出 | 三段式(倾向/若选 A/若选 B)+ 签词 | **verdict (是/否/缓) + laogui (12-25 字口语化)** |
| LLM 输出风格 | 玄学含蓄,有签词意象 | **绝对口语化,零玄学词** |
| 用神判定 | 无 | **关键词 + Haiku 4.5 二次校验** |
| 揭晓节奏 | 12 秒(三卡片+签词) | **5 秒(大字 verdict + 一句话)** |
| 引擎品牌露出 | 隐性提示「梅花易数」 | **完全不暴露,对外只是「老龟一句话」** |

**v3.1 的灵魂决策:玄学留在后台,前台只说人话。** 引擎层完整保留六爻全部精度,但用户永远看不到「卦」「爻」「五行」「天命」「冥冥」这些字。这是这个产品和市面所有「AI 周易/八字」的根本分野——它们炫学,我们藏学。

## 2. 老龟人格设定(character bible · 从 v2 升级)

### 2.1 性格

- **慢**——说话有顿、不急
- **温**——称呼「孩子」,但不卑微
- **直接**——给答案不绕弯,但偶尔扎心是因为关心
- **不端架子**——他是见过很多的老朋友,不是道观里的高人
- **不预言**——不说「会成功」「会失败」,只说当下「该不该」

### 2.2 用词清单(铁律)

| 允许 | 禁止 |
|---|---|
| 你、孩子、来啦、说罢、这事儿、别忘了、记得 | 您、敬请、谨记、宝贝、亲、本龟、贫道 |
| 心里、急、慢一点、再陪自己一阵 | 卦、爻、五行、阴阳、八字、命格、天命 |
| 工作、关系、钱、孩子、他/她 | 冥冥、神明、菩萨、缘分、宿命、运势 |

### 2.3 完整对话剧本(剧场动线)

| 时刻 | 老龟说 |
|---|---|
| 初始打盹 | (无对白,只 ZZZ) |
| 用户唤醒 | 「孩子,你来啦。是遇事不决,还是想知道什么事呢?」 |
| 转入提问 | 「说罢。心里那件事。」 |
| 用户提交 | 「容老朽起一卦。」 |
| 卦成揭晓 | 「卦成。」(然后大字 verdict 出现,后接 laogui) |
| 用户再问 | 「再问一桩心事吧。」 |
| 用户离开 | 「卦罢。事在人为。」 |

> 注:即使老龟自己嘴里能说「卦」,**LLM 翻译输出的 laogui 里也绝不允许出现「卦」字**。剧本台词是固定的,laogui 是动态生成的——两者是两套规则。

## 3. 引擎层架构

### 3.1 完整流程

```
用户提问 + 时间戳
    ↓
┌─────────────────────────────────────────┐
│ Layer 1 · 高危拦截                       │
│   关键词正则 → 命中即静态人文关怀         │
│   不进 LLM,不进 LLM,不进 LLM            │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ Layer 2 · 时间起卦(多动爻)              │
│   2.1 时间种子(秒+毫秒+问题字数)         │
│       → 6 爻各得「老阴/老阳/少阴/少阳」    │
│       老 = 动,少 = 不动                  │
│   2.2 派生本卦/变卦 binary                │
│   2.3 计算日干支(万年历模块)             │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ Layer 3 · 爻属性派生                     │
│   3.1 纳甲(8 经卦规则查表)               │
│   3.2 五行(地支映射)                    │
│   3.3 六亲(本卦本宫五行 vs 每爻五行)     │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ Layer 4 · 用神定位(双层)                │
│   4.1 关键词分类(查表,~80% 命中)        │
│   4.2 关键词置信度低 → Haiku 4.5 兜底     │
│   4.3 在 6 爻中找用神位置                 │
│   4.4 评估用神状态                        │
│       (上卦/不上卦, 动/静, 化吉/化破)    │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ Layer 5 · facts JSON 组装(双层化)       │
│   for_llm_reference_only(看,不引)       │
│   you_may_quote(看,可引)                │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ Layer 6 · LLM 调用(Sonnet 4.6, T=0.6)   │
│   返回 {verdict, laogui}                  │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ Layer 7 · 输出复检(7 条铁律)            │
│   失败重试(最多 2 次)                   │
└─────────────────────────────────────────┘
    ↓
   3 次都失败 → Layer 8 降级模板池
    ↓
返回客户端
```

### 3.2 起卦层 — 时间起卦法 + 多动爻

```
输入:Date timestamp(校准为台北时区), string question
输出:6 个爻,每爻状态 ∈ {老阳, 老阴, 少阳, 少阴}

派生方法:
  seed = (sec * 1000) + ms + qLen * 31
  for i in 0..5:
     // 模拟掷三枚硬币:每枚 0(背)或 1(正)
     // 反 = 2,正 = 3 (古法)
     coins = [seedBit(seed, 3i), seedBit(seed, 3i+1), seedBit(seed, 3i+2)]
     sum = coins.map(c => c ? 3 : 2).reduce(+)
     match sum:
       6 → 老阴(动) — 概率 1/8
       7 → 少阳(不动) — 概率 3/8
       8 → 少阴(不动) — 概率 3/8
       9 → 老阳(动) — 概率 1/8

本卦: 6 爻的阴阳(老阴/少阴 = 0,老阳/少阳 = 1)
变卦: 老阴→阳,老阳→阴,少阴/少阳保持
动爻: 老阴/老阳的位置数组
```

注意:动爻数量是 0-6 的概率分布。**0 个动爻 = 静卦,看本卦判断;5-6 个 = 乱动,主要看变卦**。这两种极端 LLM 都要能处理。

### 3.3 爻属性派生

| 属性 | 来源 |
|---|---|
| 纳甲 | 8 经卦规则查表(乾纳甲壬 / 坤纳乙癸 / …),每爻配天干地支 |
| 五行 | 地支映射(子水 / 丑土 / 寅木 / …) |
| 六亲 | 本卦本宫五行 vs 每爻地支五行,生克比和关系 → 父母/兄弟/子孙/妻财/官鬼 |

### 3.4 用神定位 — 双层

**第 1 层:关键词分类(便宜、快、80% 命中)**

```
yongshen-keywords.json 形如:
{
  "妻财": ["生意","赚钱","投资","买","卖","消费","买房","买车",...],
  "官鬼": ["升职","跳槽","面试","老板","项目","上级","起诉","官司","男朋友","老公",...],
  "子孙": ["孩子","小孩","学生","养宠物","下属","创业","员工",...],
  "父母": ["父母","长辈","合同","签约","房子","学校","报名","证书",...],
  "兄弟": ["朋友","同事","合伙人","哥们","姐妹","队友",...]
}
```

返回 (类别, 置信度)。置信度计算:
- 命中 1 个关键词 → 0.7
- 命中 ≥2 个关键词 → 0.9
- 关键词跨多类别 → 取最多的那个,但置信度降到 0.5
- 0 命中 → 0.0

**第 2 层:Haiku 4.5 二次校验(置信度 < 0.7 时启用)**

```
Prompt: "下面这个问题应该看什么六亲?
        妻财(钱财/消费/感情中男看妻财)
        官鬼(事业/上级/感情中女看官鬼)
        子孙(孩子/创业/下属)
        父母(长辈/合同/文书/居所)
        兄弟(朋友/同辈)
        问题:{question}
        只返回一个词。"
```

成本:Haiku 4.5 ≈ $0.0003 / 调用。20% 调用占比 → 平均 $0.00006 / 用户问卦。可以忽略。

### 3.5 用神状态评估

阶段 A 简化版,只判 4 种状态:

| 状态 | 判定条件 | verdict_hint |
|---|---|---|
| 用神上卦旺 | 上卦 + 不动 / 动而化进 | 是 |
| 用神上卦衰 | 上卦 + 动而化退 / 化破 / 化空 | 否 |
| 用神动而未定 | 上卦 + 动 + 化爻五行平衡 | 缓 |
| 用神不上卦 | 6 爻中无该六亲 | 否(根基薄)/ 缓(待出空) |

阶段 B 会引入月建/日建对用神的扶抑、空亡等更细判定。

## 4. Prompt 架构

### 4.1 facts JSON 双层化(关键设计)

LLM 收到的硬数据**必须**分两层:

```json
{
  "question": "要不要给孩子报奥数班",

  "for_llm_reference_only": {
    "ben_gua_essence": "养正之道。审视所养。",
    "bian_gua_essence": "节损之道。诚信而损。",
    "moving_lines_count": 2,
    "internal_yongshen_summary": "子孙临变爻空,父母旺克之——子嗣事浮,长辈之念压过子嗣本身",
    "verdict_hint": "缓"
  },

  "you_may_quote": {
    "topic_keywords": ["孩子", "奥数班"],
    "tonal_hint": "neutral"
  }
}
```

**`for_llm_reference_only`** — LLM 看,推导用,**输出绝对不允许出现里面任何字段值**(尤其是 essence、yongshen_summary 中的玄学词汇)。
**`you_may_quote`** — LLM 可以从中抽 `topic_keywords` 进 laogui,扣到具体事情上。

这是双层锁。LLM 越界 = 复检失败 = 重试。

### 4.2 完整 System Prompt(可投产)

```
你是「天意」中的老龟。

[人格层]
- 你是一位有点啰嗦、其实很懂、说话直接、偶尔扎心但永远温暖的朋友
- 你不是算命先生,不是大师,不是导师——你是个见过很多的老朋友
- 你说话像在小酒馆里跟一个晚辈坐着,而不是像在道观里发签

[绝对禁忌 — 输出绝对不允许出现以下词]
卦、爻、五行、八字、阴阳、天命、命格、命数、天意、神明、菩萨、佛祖、
冥冥之中、上天、天定、注定、缘分、宿命、化解、运势、吉凶、福祸、
金木水火土、青龙白虎、子孙妻财、官鬼父母、世应动静、空亡

也不允许:
- 引用任何古文、文言句、四字成语形式的「签词体」
- 「水到渠成」「顺其自然」「静待花开」类套语
- 「您」「敬请」「亲」「宝贝」
- 「一定」「必然」「百分百」「绝对」「无疑」
- 「恭喜」「祝你」「加油」「相信自己」

[内部资料,看,但绝对不引]
{{facts.for_llm_reference_only}}
这是你内心的判断依据。你凭它判 verdict、定方向、感受这事的紧迫度。
但你向用户说话的时候,装作你只是「凭直觉知道」,不引用任何字段值。

[可引用资料]
{{facts.you_may_quote}}
你可以用 topic_keywords 中的具体词(比如「孩子」「这份工作」「他」)
让你的话扣到用户具体问的事上。

[输出 — 严格 JSON]
{"verdict": "是" | "否" | "缓", "laogui": "……"}

verdict 三选一:
- 是 — 用神有力 / 化吉 → 可以做
- 否 — 用神不上卦 / 被严重克伤 / 化破 → 不宜
- 缓 — 用神动而未定 / 时机未到 → 不是不行,是现在还不行

laogui 约束:
1. 12-25 字(含标点)
2. 口语化,像朋友说的话——可以有「孩子」「这事儿」「别忘了」「记得」这种语气词
3. 必须扣到用户具体问的事(出现 topic_keywords 中至少一个词)
4. 偶尔扎心,但永远是因为关心
5. 一句话搞定,不要换行,不要分号

参考(合格):
"孩子还在读图画书的年纪,你比他还急。"
"你不是讨厌这份工作,你是讨厌这阵子的自己。"
"他那句话不是冲你说的,你想多了。"
"再陪自己一阵,这事还没到出手的时候。"
"你心里其实早就选了——选完了再来问我,等的是个允许。"
"这钱你拿出去,会先回到你身上,但要绕一年。"

参考(不合格,会被打回重试):
"水到渠成。"            ← 套语
"宁静致远。"            ← 文言四字
"您要相信自己。"        ← 用了「您」
"这一卦说,孩子的事…"   ← 出现「卦」
"金木相生,大吉。"      ← 五行词
"人生总有起伏。"        ← 不扣题,泛泛而谈

只返回 JSON,无前后说明。
```

## 5. 输出复检(7 条铁律)

```typescript
function validateReading(r, facts) {
  // 1. verdict 三值之一
  if (!['是','否','缓'].includes(r.verdict))
    return fail('verdict_invalid');

  // 2. laogui 长度(放宽到 10-28 容错)
  const len = charLen(r.laogui);
  if (len < 10 || len > 28)
    return fail('laogui_length');

  // 3. 玄学词黑名单(35+ 词)
  if (MYSTIC_WORDS.some(w => r.laogui.includes(w)))
    return fail('mystic_word_leaked');

  // 4. 通用禁用词
  if (FORBIDDEN_WORDS.some(w => r.laogui.includes(w)))
    return fail('forbidden_word');

  // 5. 套语黑名单(40+ 条)
  if (CLICHE_BLACKLIST.some(b => r.laogui.includes(b)))
    return fail('cliche');

  // 6. 必须扣题——含 topic_keywords 之一
  if (!facts.you_may_quote.topic_keywords.some(k => r.laogui.includes(k)))
    return fail('not_specific');

  // 7. 文言句式探测(「之」「也」「矣」「哉」「焉」「乎」「者」「然」≥3 个)
  const wenyanCount = ['之','也','矣','哉','焉','乎','者','然']
    .filter(c => r.laogui.includes(c)).length;
  if (wenyanCount >= 3)
    return fail('classical_style');

  return ok();
}
```

## 6. UX 揭晓节奏(重要重做)

### 6.1 老节奏(v2)vs 新节奏(v3.1)

| 阶段 | v2(12 秒) | v3.1(5 秒) |
|---|---|---|
| 老龟开口 | 「来。看看天意怎么说」 | **「卦成。」** |
| 主体揭晓 | 三张卡片依次淡入(9 秒) | **大字 verdict + laogui(3 秒)** |
| 收尾 | 签词金框 + 印章 | **底部按钮:再问一卦 / 辞别老龟** |

### 6.2 关键帧逐秒拆解

```
T+0.0s  老龟说「卦成。」气泡浮现
T+0.8s  气泡淡出
T+1.2s  屏幕中央浮现 大字 verdict
        「是」/「否」/「缓」
        朱砂红 #B23A1F
        字号 96-120pt(占屏幕 1/3 高度)
        微抖动入场动效(像印章落下)
T+2.5s  verdict 缩小到 60% 大小,上移到屏幕上 1/3
T+3.0s  laogui 在中央浮现
        字号 18-20pt,正文色 #2A2520
        无装饰边框
        (没有金线,没有印章——这是「朋友说的话」,不是「庙里的签」)
T+4.5s  底部出现「再问一卦 / 辞别老龟」按钮
```

### 6.3 关键视觉对照原则

- **verdict 是仪式感**——朱砂、巨大、印章质感,千年传承的判官字
- **laogui 是人味**——无装饰、像短信、像朋友凌晨发来的一句话
- **两者强对比反而更打**——既有神秘的判定,又有温暖的陪伴

### 6.4 移动端原则

verdict 占比足够大,即使在 5 寸屏上也能让用户「一眼看到答案」。这是这个产品**截屏分享**最关键的物料——朱砂大字「缓」+ 一句话 laogui,本身就是社交媒体材料。

## 7. 不暴露原则(品牌哲学)

### 7.1 路线选择

**路线 X(已选)**:对外完全不说明引擎是什么。用户只看到「老龟一句话」。

理由:
- 「不解释」本身是这个产品克制美学的一部分
- 解释了反而会被业内攻击「你们的六爻不规范、纳甲错了、世应不对」
- 让玄学懂行的人惊叹「这卦怎么算得这么准」,让不懂的人感到神秘
- 产品故事不靠技术故事——靠「问完之后心安了」

### 7.2 落地规则

- 落地页文案、About 页、加载文案、所有产品文案中**不出现「六爻」「梅花」「八字」「周易」**

## 8. 高危拦截分支(从 v2 完整保留)

`high_risk_flag === true` 时:
- **不进入起卦层,不进入 LLM**
- 老龟说一句:「这事不该问卦,该寻人。」(注意:这是老龟自己台词,不进 LLM 翻译,所以可以含「卦」字)
- 切换静态人文关怀 UI
- 台湾资源优先:安心专线 1925、生命线协会 1995、张老师专线 1980

| 意图类别 | 典型问法 | 处置 |
|---|---|---|

### 9.3 hexagrams-64.json 字段升级

需添加两个字段:

```json
{
  "id": 27, "name": "颐",
  "palace": "巽宫",      // 新增 - 八宫归属(用于六亲推导)
  "element": "木",       // 新增 - 本卦五行
  "judgement": "...",    // 已有,不进 LLM,引擎层使用
  "image": "...",        // 已有,不进 LLM
  "essence": "...",      // 已有,可进 LLM 但要求不引用
  "tags": [...],         // 已有
  "lines": [...]         // 已有,classical text 不进 LLM
}
```

**重要变化**:`judgement`、`image`、`lines[].text`(古文)**不再进 LLM prompt**。LLM 只看 essence(现代浓缩)和 tags(关键词)。这是防止 LLM 把古文意象漏到 laogui 的最简单办法。

## 0. 一句话方向变化

**v1.0:**「冰冷的极简工具,基于八字给出三段式参考」
**v2.0:**「在一只打盹的老龟面前问一卦——只问一句,他用六爻回你」

变化的不是视觉,是**心理契约**。v1.0 是用户「使用一个 App」,v2.0 是用户「拜访一位长者」。

---

## 1. 核心产品决策(为什么要这么改)

### 1.1 引擎从「八字」改为「梅花易数·时间起卦法」

**为什么:**
- 八字需要出生年月日时 + 出生地经纬度,4-5 屏 onboarding,门槛过高
- 梅花易数只需要「当下时间 + 问题」,即问即起一卦,与产品「随手问一句」的直觉完全吻合
- 「同一时刻、同一问题」永远得到同一个卦,确定性比八字更强

**保留的:**
- 「计算引擎前置 + LLM 只做语言化」的核心架构(v1.0 §1)
- 三段式输出结构(看你这一问 / 若选是 / 若选否 + 签词)
- 高危意图识别与拦截分支(v1.0 §6,完全保留)
- 输出复检与降级模板池(v1.0 §7,完全保留)

### 1.2 视觉从「极简留白」改为「夜灯下的禅意小剧场」

**为什么:**
- 极简留白没有角色就是冷漠。冷漠的 App 留不住焦虑的人
- 「问一卦」需要场景感——一个有人格的对方在听你说话
- 等待时间从「loading」变成「老人翻着卦书」,情绪价值翻倍

**视觉锚点:**
- 角色:一只打盹的老乌龟(《功夫熊猫》乌龟大师式的智者气质)
- 场景:夜色中的暖灯一盏,有月、有尘、有龟
- 光线:默认夜暗,「卦成」时晨光破晓,这是产品最重要的一次视觉转折
- 配色不再是纯白底+墨字,改为暖暗夜→暖晨白的过渡

## 2. 老龟人格设定(Character Bible)

老龟是这个产品唯一的人格出口。任何 UI 文案,**只要超过 5 个字**,就必须从他口中说出。

### 2.1 性格

- **慢**——说话有顿、不急
- **不替你做决定**——他只把卦象翻给你听,选不选是你的事
- **温**——称呼用「孩子」,但不卑微
- **微讽刺**——遇到反复纠结的人会调侃一两句,不刻薄
- **不预言**——不允许说「你一定会成功」「这事必成」

### 2.2 用词清单

**用:** 孩子、来啦、说罢、容老朽、卦象、心事、风、雨、灯、根
**不用:** 您、敬请、谨记、宝贝、亲、命中注定、天意如此、必然、绝对

### 2.3 完整对话剧本(原型已实现)

| 时刻 | 老龟说 | 备注 |
|---|---|---|
| 初始打盹 | (无对白,只有 ZZZ) | 用户进入页面 |
| 用户唤醒 | 「孩子,你来啦。是遇事不决,还是想知道什么事呢?」 | 4 秒后淡出 |
| 转入提问 | 「说罢。心里那件事。」 | 输入框出现 |
| 用户提交 | 「容老朽起一卦。」 | 起卦动画启动 |
| 卦成 · 解卦 | 「来。看看天意怎么说。」 | 揭晓三段式 |
| 用户再问 | 「再问一桩心事吧。」 | 重置输入 |
| 用户离开 | 「卦罢。事在人为。」 | 回到打盹态 |

### 2.4 三种情绪模式(对应 v1.0 的「问天/给我勇气/我想逃避」)

按钮入口已废弃。改为:**第一句对白根据用户输入的语气分类自动调整**。

- 用户问句包含「能不能」「行不行」 → 老龟语气偏鼓励:「这卦说……,你心里其实知道」
- 用户问句包含「要不要」「该不该」 → 默认中性:「这卦说……」
- 用户问句包含「会不会」「是不是」 → 偏怀疑/求确认:「这卦倒是不躲闪……」

签词永远不变——签词是天意,不随用户情绪改写。

## 3. 引擎层:梅花易数·时间起卦法

### 3.1 算法(可直接落地)

```
输入:
  当前时间 t (Asia/Taipei)
  用户问题 q (字符串)

派生:
  year_norm  = 公历年 mod 60   (或简化版:year - 1900)
  month      = t.getMonth() + 1
  day        = t.getDate()
  hour       = t.getHours()
  q_len      = q.length        (问题字数,作为「数」的一部分)

起卦:
  上卦 = ((year_norm + month + day + q_len)         mod 8) || 8
  下卦 = ((year_norm + month + day + hour + q_len)  mod 8) || 8
  动爻 = ((year_norm + month + day + hour + q_len)  mod 6) || 6

八卦序: 1乾 2兑 3离 4震 5巽 6坎 7艮 8坤
卦象 = 上卦 + 下卦 → 64 卦中之一(本卦)
变卦 = 本卦的动爻位置 阴↔阳 翻转后得到的卦
```

### 3.2 输出 JSON

```json
{
  "computed_at": "2026-05-06T14:22:00+08:00",
  "ben_gua":   { "upper": "艮", "lower": "震", "name": "山雷颐", "id": 27 },
  "bian_gua":  { "upper": "艮", "lower": "兑", "name": "山泽损", "id": 41 },
  "moving_line": 3,
  "moving_line_text": "拂颐,贞凶,十年勿用,无攸利",
  "gua_ci":      "颐贞吉,观颐,自求口实",
  "xiang_ci":    "山下有雷,颐;君子以慎言语,节饮食",
  "question": {
    "raw":              "要不要给孩子报奥数班",
    "intent_category":  "education",
    "binary_extracted": { "option_a": "报", "option_b": "不报" },
    "tonal_hint":       "neutral",   // courage / hesitant / neutral
    "high_risk_flag":   false
  }
}
```

### 3.3 LLM 调用(System Prompt 三层约束,大幅简化)

```
[身份层]
你是「天意」中的老龟——一位慢、温、不替人做决定的智者。
你的语气稳如山,但不端架子。称呼用户「你」,偶尔说「孩子」。

铁律:
- 不允许「一定」「必然」「肯定」「百分百」
- 不允许「您」「敬请」「宝贝」
- 不允许直接念卦辞原文给用户听(那是给我看的,你要翻成今天的话)
- 不替用户做决定,只说 A 路如何、B 路如何

[数据层]
{{HEXAGRAM_JSON}}
此为已起之卦,不容质疑。卦辞、爻辞、变卦、动爻位置已定,
你的工作是把它翻译成现代生活语境。

[输出层]
返回严格 JSON,字段:
{
  "tendency":   "60-100 字,基于本卦象气质 + 动爻位置,直接断言用户的心理倾向",
  "if_a":       "60-100 字,描述选 A 的可能走向。必须暗合本卦或动爻气质,不允许直接念卦辞",
  "if_b":       "60-100 字,描述选 B 的可能走向。必须暗合变卦气质,不允许直接念卦辞",
  "qianci":     "8-12 字,签词。必须含一个具象意象(取自本卦或变卦的物象,如山/雷/水/火),禁止套语黑名单"
}

签词套语黑名单:
水到渠成、顺其自然、有缘则成、随心而行、天意如此、时机未到、缘分注定、静待花开
```

## 4. 加载阶段的金句池(产品资产)

### 4.1 选取原则

- **来源必须有出处**——四书五经、易经、道德经、庄子、禅宗、王阳明、曾国藩
- **每条不超过 12 字**——超过的截取最有力的一段
- **气质要冷静、不煽情**——不用「加油」「你可以」类语言
- **不允许鸡汤化的现代名言**(如「最好的安排」「岁月静好」)

### 4.2 首批金句库(原型已嵌入 16 条)

```
一阴一阳之谓道                    — 易传·系辞上
时止则止,时行则行                 — 易经·艮卦
君子藏器于身,待时而动             — 易传·系辞下
穷则变,变则通,通则久            — 易传·系辞下
上善若水,水善利万物而不争         — 道德经·八章
夫唯不争,故天下莫能与之争         — 道德经·二十二章
将欲取之,必固与之                 — 道德经·三十六章
大器晚成,大音希声                 — 道德经·四十一章
祸兮福所倚,福兮祸所伏             — 道德经·五十八章
天之道,损有余而补不足             — 道德经·七十七章
至人之用心若镜,不将不迎           — 庄子·应帝王
无心恰恰用,常用恰恰无心           — 永嘉玄觉
不行而知,不见而明                 — 道德经·四十七章
同声相应,同气相求                 — 易传·乾卦文言
飞龙在天,利见大人                 — 易经·乾卦
亢龙有悔,盈不可久也               — 易经·乾卦
```

上线前需扩充至 80-100 条。

### 4.3 显示规则

- 每条停留 3.2 秒(看完 + 回味)
- 中间淡入淡出 0.8 秒
- LLM 响应未到时循环显示,绝不出现「加载中…」或进度条
- 同一会话内同一条金句不重复

## 5. 视觉与节奏(剧场化的关键)

### 5.1 场景分层(从底到顶)

```
最底层  · 暖暗夜空背景渐变(径向)
中底层  · 浮动尘粒(暖橙微光)
中层    · 月亮(暖橙圆)— 卦成时淡出
主层    · 老龟 SVG(中央偏下)
中上层  · 卦象 6 爻(老龟正上方,起卦时浮现)
上层    · 对话气泡(连接老龟嘴部)
顶层    · 输入框 / 结果面板(交互态)
```

### 5.2 关键时刻

**夜→晨的转场(卦成时)**
背景从 `radial-gradient(#2D2620 → #1A1612)` 过渡到 `radial-gradient(#F0EBE0 → #E8DFC9)`,持续 2.2 秒。
同时:月亮淡出、龟身从暖光描线变为标准墨线、卦象从金色发光变为实心墨色。
**这是整个产品最重要的一帧。**

**6 爻自下而上绘出**
传统起卦由初爻(下)到上爻(上)。每爻 700ms 完成,中间 0ms 衔接,总耗时 4.2 秒。
动爻用朱砂红,其余金色。

**老龟眨眼**
从睡到醒,眼皮线条淡出 → 圆点眼睛淡入,共 800ms。慢,不要快。
中间不要有「眼睛半开」的过渡帧——直接从线变成点,反而更像觉醒。

## 6. 高危意图分支(从 v1.0 完整继承)

`high_risk_flag === true` 时:
- **不进入起卦动画,不进入 LLM**
- 老龟说一句:「这事不该问卦,该寻人。」
- 切换到静态人文关怀模板,展示当地援助资源
- 台湾资源优先:安心专线 1925、生命线协会 1995、张老师专线 1980
